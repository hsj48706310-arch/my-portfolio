Project Context & Execution

기존 문제 (Problem)

유연성 부족 및 비용 불명확

7개 지역과 10개 테마 조건의 실시간 결합 요구가 증가했으나, 복잡한 입력을 반영한 코스 추천과 정밀한 비용 계산 수식을 처리할 지능형 아키텍처 부재.

맡은 업무 (Role)

아키텍처 설계 & 엔진 개발

LangGraph 프레임워크를 도입하여 복잡한 요구사항을 상태 단위로 나누고, 프론트엔드 팀원과 연동할 데이터 규격(Spec) 선제적 정의 및 코어 엔진 개발 총괄.

진행 사항 (Action)

워크플로우 최적화 및 파이프라인

'입력 수집 → 추천 → 비용 계산 → 요약'의 유연한 순환 파이프라인 설계. travel_data.py 상에 정밀 수식 구현 및 JSON 파싱 에러 방지용 Fallback 메커니즘 연동.

Project Outcome

결과 (Result)

수동 계획 대비 50% 이상의 효율 개선

복잡했던 여행 계획 수립 과정을 단 7단계의 자동화 입력 인터페이스로 단축 완료.

네트워크 및 LLM API 순간 오류 발생 환경 속에서도 다운타임 없이 가동되는 강력한 시스템 가용성(Crash-free)을 실현하며 프로젝트 완성도를 입증했습니다.
